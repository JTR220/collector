- **fix(security)**: bump Go `1.26.4` → `1.26.5` sur les 4 services backend — govulncheck bloquait le pipeline sur une fuite de confidentialité ECH dans `crypto/tls` (GO-2026-5856) (commit `0470604`).

- **feat(catalog)**: upload de photo réintroduit (`POST /article/:id/image`), avec les garde-fous qui manquaient à l'ancienne version retirée lors du précédent durcissement sécurité :
  - type réel du fichier vérifié par ses premiers octets (jamais l'extension ni le Content-Type déclarés par le client)
  - nom de fichier généré côté serveur (UUID) via `os.Root` (Go ≥1.24) — élimine tout risque de path traversal par construction, pas seulement "on fait confiance à l'UUID"
  - limite 5 Mo, formats jpeg/png/webp/gif uniquement
  - dossier `/uploads` servi en pur statique, `X-Content-Type-Options: nosniff`
  - (commits `4beefe6`, `97b7364` pour le fix gosec G304/G302 associé)
